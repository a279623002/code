# RAG 检索增强生成

> 一句话：RAG 让大模型先查资料再回答，解决知识滞后、私有数据和幻觉问题。

---

## 一、什么是 RAG？

**一句话**：RAG 是"先检索、后生成"的架构，给大模型补充外部知识。

### 解决的痛点

| 痛点 | 说明 |
|---|---|
| 知识滞后 | 模型训练数据有截止时间 |
| 私有数据 | 企业知识不在训练集中 |
| 幻觉 | 模型容易编造答案 |

---

## 二、RAG 完整链路

```
离线阶段：
文档 → 清洗 → 分片 → Embedding → 向量库存储

在线阶段：
用户提问 → 检索召回 → Rerank 精排 → 上下文组装 → LLM 生成答案
```

---

## 三、核心模块

### 1. 文档分片

| 策略 | 优点 | 缺点 |
|---|---|---|
| 固定长度 | 简单快速 | 容易切断语义 |
| 语义分片 | 语义完整 | 成本高 |
| 重叠分片 | 避免信息断裂 | 存储增加 |

**常用做法**：固定长度 + 重叠窗口 + 按标题边界优化。

### 2. 检索召回

| 方式 | 原理 | 优点 | 缺点 |
|---|---|---|---|
| **向量检索** | 语义相似度 | 泛化能力强 | 精准词易漏召 |
| **关键词检索（BM25）** | 字面匹配 | 精准名词匹配好 | 口语化场景弱 |
| **混合检索** | 两者结合 | 覆盖全面 | 实现稍复杂 |

### 3. Rerank 重排

**一句话**：Rerank 是对召回结果做二次精排，过滤噪声，提升相关性。

```
召回 20 条 → Rerank 打分 → 取 Top 5 → 送入 LLM
```

### 4. 上下文组装

- 去重
- 按相关度排序
- 控制 Token 总量
- 关键信息置顶

## 四、面试高频问题

### Q1：RAG 解决了什么问题？

**答**：解决大模型知识滞后、无法访问私有数据、容易幻觉的问题，让模型基于检索到的真实资料回答。

### Q2：向量检索和关键词检索的区别？

**答**：向量检索基于语义相似度，适合模糊、口语化提问；关键词检索基于字面匹配，适合专有名词、编号等精准查询。

### Q3：为什么要用混合检索？

**答**：单一检索有短板。向量检索对精准词可能漏召，关键词检索对语义改写可能漏召。混合检索互补，提高召回覆盖率。

### Q4：Rerank 的作用是什么？

**答**：粗召回保覆盖，Rerank 保精准。Rerank 对召回结果做二次打分，过滤低相关噪声。

### Q5：怎么评估 RAG 效果？

**答**：召回层看召回率、精准率；生成层看答案准确率、忠实度、无幻觉率。

## 五、项目口述模板

> 请介绍你项目中的 RAG 方案。

我在项目中落地了完整的 RAG 链路。离线阶段对文档做清洗、分片、Embedding 向量化，存入向量库；在线阶段用户提问时，先用向量 + BM25 混合检索召回候选片段，再用 Rerank 模型精排过滤噪声，最后将 Top 相关片段组装进 Prompt，由 LLM 生成答案。针对切片做了固定长度 + 重叠窗口 + 标题边界优化；针对检索做了 Query 改写和混合召回；针对生成做了忠实度约束，禁止无依据编造。

---

## 六、一句话总结

- **RAG = 检索 + 生成**
- **向量检索泛化强，关键词检索精准高，混合检索最常用**
- **Rerank 是性价比最高的精度提升手段**
- **效果评估：召回率 + 精准率 + 答案忠实度**
