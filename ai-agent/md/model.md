## 二、Agent开发专项场景选型对比（新增DeepSeek、千问）
### 场景1：长文档解读Agent（合同、财报、论文、上万字资料）
1. **最优：Claude > DeepSeek V4**
    Claude原生200K/1M窗口，长文本事实准确率最高；DeepSeek V4百万上下文，国产私有化长文档首选；
2. **次选：GPT-4o / Qwen Max（128K）**
    128K窗口满足大部分常规文档，但十万字以上易信息稀释；
3. **不推荐原生LLaMA**
    原生仅8K窗口，必须分块RAG、摘要记忆二次加工，开发工作量翻倍。

### 场景2：图文多模态Agent（截图分析、图表识别、OCR+推理、视频解读）
1. **最优：GPT-4o**
    图文音视频全模态一体化，工具调用联动视觉；
2. **次选：通义千问Qwen-VL**
    国产唯一原生支持视频+工具调用，国内合规、成本更低；
3. **Claude 仅支持静态图片，无视频能力；LLaMA/DeepSeek无原生多模态，需额外搭视觉模型。**

### 场景3：企业涉密内网Agent（金融、政务、数据禁止出内网、国内合规）
1. **首选分两类：**
    - 追求中文行业适配、快速落地：**通义千问Qwen**，国内生态完善、合规对齐；
    - 追求代码/数学推理、百万长文本：**DeepSeek V4**；
    - 通用低成本私有化：**LLaMA3**；
2. **GPT / Claude 直接淘汰**
    海外公有云API存在数据出境、国内合规风险，无法满足等保、涉密要求。

### 场景4：代码/数据分析自动化Agent（代码库解析、SQL查询、数据报表、批量计算）
1. **最优梯队：Claude > DeepSeek R1/V4 > GPT-4o > QwQ > LLaMA**
    DeepSeek专项优化代码推理，百万行代码库一次性解析，多步骤数据计算自主规划；Qwen配套Qwen-Agent代码解释器开箱即用。

### 场景5：高并发大批量调用、控制Token成本、中小企业轻量化Agent
1. **最优：开源私有化（Qwen / DeepSeek / LLaMA）**
    一次性硬件投入，海量调用无额外Token费；
2. **公有云低成本备选：通义千问阿里云API、DeepSeek国内API**
    单价远低于GPT/Claude，适合中小团队快速上线。

### 场景6：多工具、多步骤规划Agent（数据库查询、计算、报表生成、链式调用、多智能体协作）
1. **梯队排序：Claude > GPT > Qwen > DeepSeek > LLaMA**
    Qwen配套官方Qwen-Agent框架，MCP多智能体协议原生支持；DeepSeek支持工具失败自动重试；LLaMA必须Tool-RAG+工具SFT微调才能稳定使用。

## 三、面试高频优缺点标准答案（直接背诵，新增DeepSeek、Qwen）
### 1. GPT 做Agent优缺点
**优点**
1. Function Calling生态最成熟，社区工具、框架（LangChain/LlamaIndex）适配完善；
2. 多模态能力行业顶尖，图文音视频一体化Agent首选；
3. 通用推理能力强，通用场景开箱即用，无需微调；

**缺点**
1. 数据走海外公有云，国内涉密、政务企业无法使用；
2. 海量并发调用Token成本高；
3. 超长上下文弱于Claude/DeepSeek；
4. 无法本地私有化微调，定制化能力受限；
5. 中文行业术语、国内业务场景适配一般。

### 2. Claude 做Agent优缺点
**优点**
1. 超长上下文窗口，长文档、法律合同类Agent天花板；
2. 结构化输出、工具调用稳定性五款模型最强，多轮Agent容错高；
3. 长文本事实幻觉最低，金融、审计、长篇报告可靠性高；

**缺点**
1. 无开源权重，只能依赖海外云端API，国内数据合规风险大；
2. 不支持视频、音频多模态；
3. 短对话场景Token单价略高于GPT；
4. 中文本土化优化不足，国内政企场景适配弱于DeepSeek/Qwen。

### 3. LLaMA 系列做Agent优缺点
**优点**
1. 全球开源标杆，生态成熟，各类Agent框架全面适配；
2. 开源可本地离线部署，数据安全可控，适配海外涉密场景；
3. 支持LoRA/QLoRA低成本微调，可定制专属工具调用；
4. 海外社区资料极多，学习资料丰富。

**缺点**
1. 原生无官方Function Calling，必须额外微调才能稳定调用工具；
2. 幻觉严重，事实生成容易编造数据；
3. 长上下文原生窗口仅8K，需要大量工程优化；
4. 中文原生优化薄弱，国内业务落地效果差；
5. 原生安全防护弱，易遭受Prompt注入攻击。

### 4. DeepSeek（V4/R1）做Agent优缺点
**优点**
1. 国产开源，支持百万字超长上下文，代码/数学推理开源第一梯队；
2. 原生兼容OpenAI标准Function Calling，多步骤工具调用支持自动回溯纠错；
3. 国内合规公有云API+开源私有化双部署模式，数据可不出国内；
4. MoE混合专家架构，推理性价比高，高并发显存占用更低；
5. 中文代码、技术文档、数值计算场景表现优异。

**缺点**
1. 事实类文档幻觉中等偏上，容易生成逻辑自洽的错误结论；
2. 无原生多模态能力，图文/视频Agent需要额外集成视觉模型；
3. 通用行业对齐弱于通义千问，政企、电商等场景需额外微调；
4. 安全防护基础，内网私有化必须自行增加权限校验层。

### 5. 通义千问Qwen2.5/QwQ 做Agent优缺点
**优点**
1. 国内全栈适配最优，原生中文专项训练，政务、金融、电商行业开箱即用；
2. 开源+阿里云合规云双路线，完全满足国内等保、数据不出境要求；
3. 原生全模态（图文音视频），Qwen-VL支持视觉+工具联动，国内多模态Agent首选；
4. 配套官方开源Qwen-Agent智能体框架，MCP多智能体协议原生兼容，开发成本极低；
5. 标准化Function Calling，支持自定义JSON Schema，并行多工具调用稳定；
6. 微调流水线成熟，阿里云百炼提供可视化LoRA微调，中小企业易落地；
7. 安全护栏符合国内法规，内置Prompt注入防护、工具黑白名单权限管控。

**缺点**
1. 百万超长上下文能力弱于DeepSeek V4、Claude Opus；
2. 纯英文、海外专业文档推理略逊于GPT/Claude；
3. 开源7B小参数复杂多步推理弱于70B LLaMA/DeepSeek大版本。

## 四、面试场景题标准答题模板（国内企业面试高频）
> 题目：搭建国内企业业务Agent，可选择GPT/Claude/LLaMA/DeepSeek/通义千问，如何选型？
1. **先判断数据合规与地域要求（国内面试必答）**
    - 客户数据禁止出境、政企/金融涉密：直接淘汰GPT、Claude；在DeepSeek、Qwen、LLaMA三者选择；
    - 海外C端产品无合规限制：可选GPT/Claude；
2. **区分业务场景**
    - 业务包含截图、视频、OCR多模态交互 → 优先通义千问Qwen-VL；
    - 业务以百万行代码库、数值计算、数据分析自动化为主 → 优先DeepSeek V4/R1；
    - 长篇合同、财报、百万字文档解析，且允许公有云 → 选Claude；
    - 轻量化内部工具Agent、预算有限、高并发调用 → 开源私有化Qwen；
    - 海外业务、通用多模态对话 → GPT-4o；
3. **成本权衡**
    - 每日百万级调用、长期成本敏感：开源私有化Qwen/DeepSeek/LLaMA；
    - 少量低频快速上线、国内合规：DeepSeek或通义千问公有云API；
    - 少量海外用户、不计成本：GPT/Claude；
4. **落地配套补充**
    - 选DeepSeek：重点优化幻觉，搭配RAG做事实校验；无多模态需额外集成视觉模型；
    - 选通义千问：直接使用官方Qwen-Agent框架，开箱即用，中文行业微调资料丰富；
    - 选LLaMA：必须做工具调用专项SFT微调、长文本RoPE拓展、安全校验层；
    - 选GPT/Claude：仅适合公网无涉密业务，依赖原生Function Calling快速开发。

## 五、高频追问标准答案（新增DeepSeek、Qwen相关问题）
### 1. LLaMA原生工具调用差怎么解决？
① 构造工具调用轨迹数据集做SFT微调；
② 使用QLoRA轻量化专项优化工具输出格式；
③ Tool-RAG动态筛选少量工具放入上下文；
④ 输出增加正则/JSON校验，失败自动重试；
⑤ 增加强约束Few-shot示例。

### 2. 国内私有化Agent，DeepSeek和通义千问怎么选？
- 代码、数学、百万字超长文档、MoE高并发推理需求 → DeepSeek；
- 多模态图文/视频、国内政企/电商行业、快速开发、配套完整Agent框架 → 通义千问Qwen。

### 3. 超长文档Agent为什么Claude优于GPT，国产选DeepSeek？
Claude原生200K/1M上下文，长文本首尾信息一致性、事实准确率业内顶尖；GPT 128K窗口接近上限易信息稀释；DeepSeek V4是国产唯一原生支持1M上下文的开源模型，满足国内数据不出境需求。

### 4. 通义千问Qwen做Agent对比DeepSeek有什么独特优势？
① 原生全模态图文视频工具联动，DeepSeek无原生多模态；
② 官方配套Qwen-Agent完整开发框架，MCP多智能体原生支持，DeepSeek无配套官方Agent脚手架；
③ 国内中文政企、电商场景原生对齐，幻觉更低，安全合规符合国内法规；
④ 阿里云百炼提供一站式微调、部署、监控平台，中小企业落地门槛更低。

### 5. 私有化LLaMA对比DeepSeek/通义千问，国内工程落地劣势？
LLaMA中文原生优化差，无国内合规云服务，无配套Agent开发框架，长文本拓展、工具调用、安全防护都需要从零开发；DeepSeek、Qwen针对国内场景做全链路优化，提供API、微调、Agent框架完整配套。

### 6. DeepSeek幻觉问题如何缓解？
① 强制标签分步推理，约束模型输出思考过程；
② 接入RAG检索真实知识库，工具返回结果做事实校验；
③ 专项SFT微调，增加幻觉纠正负样本；
④ 增加后置校验层，对数值、合同、财务数据做正则/数据库二次核对。
