# Skill 注册发现执行框架

> 一句话：Skill 是 Agent 的业务能力单元，框架负责注册、发现、执行三步闭环。

---

## 一、Skill 框架总览

```
┌─────────┐    ┌─────────┐    ┌─────────┐
│  注册   │ → │  发现   │ → │  执行   │
│ Register│    │ Discover│    │ Execute │
└─────────┘    └─────────┘    └─────────┘
```

| 阶段 | 作用 |
|---|---|
| **注册** | 把 Skill 元数据标准化入库 |
| **发现** | 运行时匹配用户意图到 Skill |
| **执行** | 调度 Plugin，完成业务逻辑 |

---

## 二、Skill 注册

### 1. 注册内容

```
Skill 元数据 = {
  id: 唯一标识,
  name: 技能名称,
  description: 功能描述,
  triggers: 触发规则,
  params: 入参 Schema,
  plugins: 依赖插件,
  priority: 优先级,
  timeout: 超时时间,
  status: 启用/禁用/灰度
}
```

### 2. 注册方式

| 方式 | 说明 |
|---|---|
| **静态注册** | 启动时加载配置文件 |
| **动态注册** | 运行时通过管理后台新增 |
| **热更新** | 不停机更新 Skill |

---

## 三、Skill 发现

### 1. 发现流程

```
用户输入
   │
   ▼
规则初筛（关键词、标签、黑白名单）
   │
   ▼
语义匹配（向量相似度）
   │
   ▼
冲突仲裁（优先级、场景专属度）
   │
   ▼
命中唯一 Skill
```

### 2. 冲突解决

| 问题 | 方案 |
|---|---|
| 多 Skill 命中 | 优先级 + 场景专属度 + 上下文锁定 |
| 误触发 | 强化 description 边界、配置互斥 |
| 漏触发 | Query 改写、语义扩展 |

---

## 四、Skill 执行

### 1. 执行链路

```
请求进入
   │
   ▼
参数解析（从用户输入提取参数）
   │
   ▼
参数校验（类型、必填、范围）
   │
   ▼
缺参追问（必要时询问用户）
   │
   ▼
Plugin 链式调度
   │
   ▼
结果聚合与容错
   │
   ▼
结果封装与输出
```

### 2. 渐进式加载

**一句话**：启动只加载 Skill 索引，命中后才加载完整规则。

```
传统方式：所有 Skill 全塞进 Prompt → Token 爆炸、互相干扰
渐进式：只加载索引 → 命中后加载完整 Skill → 省 Token、准确率高
```

## 五、面试高频问题

### Q1：Skill 框架包含哪些阶段？

**答**：注册、发现、执行。注册把 Skill 标准化入库；发现根据用户意图匹配 Skill；执行调度 Plugin 完成业务逻辑。

### Q2：Skill 发现怎么做？

**答**：规则初筛 + 语义匹配 + 冲突仲裁。先用关键词和标签快速过滤，再用向量相似度精排，最后用优先级和上下文锁定解决冲突。

### Q3：渐进式加载解决什么问题？

**答**：解决所有 Skill 全塞进 Prompt 导致的 Token 爆炸、上下文溢出、技能互相干扰问题。

### Q4：Skill 和 Plugin 是什么关系？

**答**：Skill 是业务场景封装，Plugin 是原子工具。一个 Skill 可以调用多个 Plugin。

### Q5：缺参时怎么办？

**答**：根据 Skill 的入参 Schema，自动识别缺失参数，用固定话术追问用户补齐。

## 六、项目口述模板

> 请介绍你项目中的 Skill 框架。

我项目中落地了完整的 Skill 注册发现执行框架。注册阶段把 Skill 元数据标准化，支持动态热注册和灰度启停；发现阶段采用规则初筛 + 语义匹配 + 优先级仲裁的三级路由，保证用户意图精准命中；执行阶段做参数解析、校验、缺参追问、Plugin 调度、结果聚合和异常兜底。同时采用渐进式加载，只预加载轻量索引，命中后才加载完整规则，大幅降低 Token 开销，避免技能互相污染。

## 七、一句话总结

- **Skill 框架 = 注册 + 发现 + 执行**
- **发现 = 规则初筛 + 语义匹配 + 冲突仲裁**
- **执行 = 参数解析 + 校验 + 调度 + 聚合 + 兜底**
- **渐进式加载 = 省 Token、提精度、易治理**
