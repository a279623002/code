## 一、基础总览面试题

### 1、Agent场景中，HTTP、WebSocket、MCP、A2A 分别承担什么定位？

**标准答案**：

在Agent体系里四类协议分工非常明确：HTTP是**基础短连接调用协议**，用于单次工具、接口、服务调用；WebSocket是**长连接实时协议**，用于流式输出、实时对话、持续任务推送；MCP是**机器控制协议**，主打Agent与工具、环境、设备的标准化交互；A2A是**多智能体协作协议**，专门用于Agent和Agent之间的通信、任务分发、协同调度。

## 二、HTTP 协议（Agent 基础通信）

### 2、HTTP 协议在 Agent 中的应用场景？核心特点？

**标准答案**：

HTTP是Agent最基础的通信方式，适用于**单次、被动、点对点**的工具调用和服务请求，比如调用外部API、查询数据库、请求第三方工具、触发单次任务。

核心特点是短连接、无状态、一问一答、简单可靠。每次调用都会建立连接、请求、响应、断开，服务端不保存客户端状态，天然适配无状态的单次Function Call。

### 3、HTTP 用于 Agent 工具调用的优缺点？

**标准答案**：

优点：接入简单、通用性极强、天然兼容所有后端接口、可直接鉴权、日志好排查、适配绝大多数工具服务。

缺点：无状态不支持持续会话、无法主动推送、多次多轮调用会反复建连、开销大、不支持流式实时返回，长任务、多轮连续任务体验差。

### 4、为什么复杂 Agent 不能只用 HTTP？

**标准答案**：

因为HTTP是单向短连接，只能客户端主动请求、服务端被动响应。而复杂Agent需要**多轮迭代、实时流式输出、任务进度推送、服务端主动回调**，HTTP无法满足实时性和持续性，必须搭配WebSocket或长连接协议使用。

## 三、WebSocket 协议（Agent 实时流式通信）

### 5、WebSocket 核心原理和 Agent 落地场景？

**标准答案**：

WebSocket是基于HTTP握手升级的**全双工长连接协议**，一次握手、全程通信，客户端和服务端可以双向主动收发数据。

Agent中主要用于：大模型流式输出、实时对话打字效果、长任务进度实时推送、多轮持续会话、Agent实时状态同步、实时日志推送。

### 6、WebSocket 和 HTTP 在 Agent 场景的核心区别？

**标准答案**：

第一，连接模式不同：HTTP是短连接、每次请求重建连；WebSocket是长连接，一次握手持久通信。

第二，通信方向不同：HTTP只能客户端主动请求；WebSocket全双工双向可主动推送。

第三，状态不同：HTTP无状态；WebSocket可维持会话状态，绑定单次Agent任务上下文。

第四，场景不同：HTTP适合单次工具调用；WebSocket适合实时流式、长会话、持续任务迭代。

### 7、WebSocket 长连接在 Agent 工程中的常见问题与处理方案？

**标准答案**：

常见问题：连接超时断开、心跳断连、消息乱序、多并发连接积压、连接泄露。

工程方案：配置心跳检测自动重连、消息序号排序、任务唯一ID绑定连接、超时自动销毁、连接池管控、断线重续上下文恢复。

## 四、MCP 协议（Machine Control Protocol 机器控制协议）

### 8、什么是 MCP 协议？核心定位是什么？

**标准答案**：

MCP 是面向AI Agent的**机器控制与工具交互协议**，核心定位是标准化「Agent调用外部工具、操作系统、设备、服务」的交互规范，解决不同工具、不同环境调用不统一的问题。

简单说：**HTTP负责通用接口调用，MCP负责Agent可控、标准化、可审计的工具与环境操控**。

### 9、MCP 协议核心优势、解决了什么痛点？

**标准答案**：

第一，标准化统一收口，各类工具、脚本、设备、API统一用MCP规范交互，不用适配多套格式；

第二，强可控性，支持权限校验、操作审计、指令拦截、参数校验，适合高可控要求的Agent场景；

第三，结构化指令交互，天然适配LLM结构化输出，适配Function Call调用范式；

第四，支持任务链路追踪，每一次工具操作可溯源、可回放、可复盘，便于BadCase迭代。

### 10、MCP 和普通 HTTP 工具调用的区别？

**标准答案**：

HTTP侧重通用数据请求，只做简单响应；MCP是**面向Agent行为控制的专用协议**，自带指令规范、权限、审计、容错、任务状态管理，专门服务于Agent自主操控外部环境，更适合自动化、高可控、可追溯的智能体任务。

## 五、A2A 协议（Agent to Agent 多智能体协议）

### 11、什么是 A2A 协议？核心作用？

**标准答案**：

A2A 是**多智能体之间的协同通信协议**，专门用于Agent与Agent之间的消息交互、任务分发、能力调用、结果同步、状态协商。

如果说MCP是Agent对外调用工具的协议，那A2A就是**Agent对内互相协作的协议**，是多智能体系统的通信基石。

### 12、A2A 协议主要解决什么场景问题？

**标准答案**：

解决多Agent架构下的协同问题：主Agent拆解任务、分发子任务给子Agent；子Agent执行完成后回传结果；多Agent之间状态同步、异常互通、任务接力、冲突协商、能力注册与发现。

### 13、A2A 和普通接口调用的本质区别？

**标准答案**：

普通HTTP调用是被动接口请求，无任务语义；A2A是**带任务语义的智能体通信**，消息体自带任务ID、角色身份、任务阶段、协作状态、执行结果、异常信息，支持自主协商、自动重试、任务接力，是面向多智能体协作的结构化通信规范。

## 六、四大协议面试对比压轴题（高频必问）

### 14、请对比 HTTP / WebSocket / MCP / A2A 四大协议选型？

**标准答案**：

**HTTP**：短连接、无状态、单向；适合单次、独立、无持续交互的工具API调用；

**WebSocket**：长连接、全双工、实时；适合流式输出、实时对话、长任务持续推送；

**MCP**：Agent\-to\-工具/环境、标准化可控；适合需要审计、权限、可追溯、自动化的工具操控场景；

**A2A**：Agent\-to\-Agent、协同语义；适合多智能体任务分发、协作、子任务调度。

### 15、多智能体系统的完整通信架构你怎么设计？

**标准答案**：

我会采用四层分层通信架构：

用户与Agent层用**WebSocket**，保证实时流式对话体验；

Agent与外部工具、设备、系统层用**MCP**，统一标准化操控、审计与权限管控；

Agent与Agent之间协作用**A2A**，实现任务分发、协同接力；

基础通用第三方接口兜底用**HTTP**，保证通用性和兼容性。

### 16、项目口述题：你在Agent项目中如何做协议选型？

**口述模版（直接背）**：

我在搭建Agent系统时，根据场景分层选型协议。用户交互侧使用WebSocket，实现流式输出和实时对话，提升用户体验；工具调用侧统一基于MCP协议做标准化封装，统一指令格式、权限校验和操作审计，解决多工具适配混乱、不可追溯的问题；多智能体协作场景采用A2A协议，实现主从Agent的任务分发和结果同步；第三方通用接口则保留HTTP兜底，保证整体系统的兼容性和通用性。通过四层协议配合，兼顾了实时性、标准化、可控性和协作能力。
