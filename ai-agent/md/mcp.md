# MCP 接口适配层

> 一句话：MCP 是 Agent 的对外统一网关，外部系统通过它调用内部 Skill/Plugin，内部迭代不影响外部。

---

## 一、为什么需要 MCP 适配层？

### 直接暴露内部能力的问题

| 问题 | 说明 |
|---|---|
| 接口不稳定 | 内部 Skill/Plugin 迭代频繁，外部跟着改 |
| 接入混乱 | 多系统对接没有统一鉴权、限流 |
| 协议不统一 | 内部能力格式各异，接入成本高 |
| 无法治理 | 接口爆炸，难以维护 |

### MCP 的核心价值

```
外部系统 ──► MCP 适配层 ──► Agent 内部能力
              │
              ├── 统一协议
              ├── 统一鉴权
              ├── 统一限流
              ├── 统一日志
              └── 统一返回格式
```

**一句话**：MCP 让内部能力快速迭代，外部接口长期稳定。

## 二、MCP 五层架构

```
┌─────────────────────────────────────┐
│  外部接入层  │  HTTP / WebSocket     │
├─────────────────────────────────────┤
│  协议网关层  │  鉴权、限流、熔断、日志  │
├─────────────────────────────────────┤
│  能力路由层  │  按能力 ID 路由到 Skill │
├─────────────────────────────────────┤
│  参数转换层  │  外部协议 ↔ 内部参数    │
├─────────────────────────────────────┤
│  能力执行层  │  Skill / Plugin 执行   │
└─────────────────────────────────────┘
```

| 层级 | 职责 |
|---|---|
| 外部接入层 | 承接第三方流量 |
| 协议网关层 | 统一治理 |
| 能力路由层 | 匹配内部能力 |
| 参数转换层 | 协议双向转换 |
| 能力执行层 | 执行具体能力 |

## 三、对外两个核心接口

### 1. 能力发现接口

```
GET /mcp/discover

返回：
[
  {
    "id": "query_order",
    "name": "查询订单",
    "description": "根据订单号查询订单状态",
    "version": "1.0",
    "schema": {...}
  }
]
```

### 2. 能力执行接口

```
POST /mcp/execute

请求：
{
  "capability_id": "query_order",
  "version": "1.0",
  "request_id": "uuid",
  "params": {"order_no": "12345"}
}

响应：
{
  "code": 0,
  "message": "success",
  "data": {...},
  "cost_ms": 120
}
```

**一句话**：两个通用接口覆盖所有能力开放场景，新增能力无需改接口。

## 四、高可用保障

| 维度 | 方案 |
|---|---|
| 流量治理 | 限流、熔断、IP 白名单 |
| 故障隔离 | 单 Skill 故障不影响全局 |
| 超时重试 | 统一超时 + 有限重试 |
| 降级兜底 | 繁忙时返回标准化提示 |
| 可观测 | 全链路日志、监控、告警 |

## 五、面试高频问题

### Q1：什么是 MCP 适配层？

**答**：MCP 是 Agent 的对外统一网关，对外暴露标准化接口，对内收敛 Skill/Plugin 能力。

### Q2：为什么不直接暴露 Skill/Plugin？

**答**：内部迭代频繁、协议不统一、缺少统一治理。MCP 让内部能力快速迭代，外部接口长期稳定。

### Q3：MCP 对外暴露几个接口？

**答**：两个：能力发现接口和能力执行接口。新增能力通过配置注册，不需要新增接口。

### Q4：怎么保证高可用？

**答**：限流、熔断、超时重试、故障隔离、降级兜底、全链路监控。

### Q5：内外参数不一致怎么办？

**答**：在参数转换层做双向适配，外部请求转内部参数，内部结果转外部标准返回。

## 六、项目口述模板

> 请介绍你项目中的 MCP 接口适配层。

我在项目中设计了 MCP 统一接口适配层，作为 Agent 对外的唯一网关。对外只暴露能力发现和能力执行两个通用接口，外部系统通过能力 ID 调用内部 Skill/Plugin；对内收敛所有 Agent 能力，屏蔽内部复杂调度。网关层统一做鉴权、限流、熔断、超时重试和日志监控，单能力故障不影响全局。参数转换层负责内外协议双向适配，保证内部随便迭代，外部接口稳定不变。

## 七、一句话总结

- **MCP = Agent 的对外统一网关**
- **核心价值：外稳内活**
- **两个接口：发现 + 执行**
- **高可用：限流、熔断、隔离、兜底、监控**
