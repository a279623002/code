# AI应用开发场景面试题

## 项目场景题

### 场景1：如何设计一个企业级知识库问答系统？

#### 问题描述
设计一个基于RAG的企业级知识库问答系统，支持多格式文档上传、智能分块、向量检索、多轮对话等功能。

#### 设计思路
1. **架构设计**
   - 分层架构：API层 → Service层 → Model层
   - 前后端分离
   - 微服务可选

2. **核心模块**
   - 文档处理模块：多格式解析、智能分块
   - 向量检索模块：FAISS存储和检索
   - LLM问答模块：调用大模型生成回答
   - 对话记忆模块：管理多轮对话
   - 模型管理模块：预加载和缓存

3. **技术选型**
   - 后端框架：FastAPI
   - AI框架：LangChain
   - 向量数据库：FAISS
   - Embedding：bge-large-zh-v1.5
   - LLM：GLM或OpenAI兼容

4. **性能优化**
   - 模型预加载
   - 向量库缓存
   - 批处理插入
   - 冷启动优化

5. **扩展性设计**
   - Loader工厂模式
   - 配置化参数
   - 多设备支持

### 场景2：如何处理大文件上传和处理？

#### 问题描述
用户上传100MB+的大文档，如何避免内存溢出并高效处理？

#### 解决方案
1. **流式处理**
   - 使用流式上传，避免全量加载内存
   - 边读取边处理，分批次处理文档

2. **分批处理**
   - 设置batch_size=50
   - 每个批次处理后及时释放内存
   - 实时记录处理进度

3. **临时文件**
   - 使用临时文件存储中间结果
   - 处理完成后清理

4. **异步处理**
   - 大文件异步处理
   - 返回任务ID，用户可查询进度

5. **资源限制**
   - 文件大小限制
   - 并发上传限制
   - 超时设置

### 场景3：如何优化RAG检索效果？

#### 问题描述
RAG系统检索结果不相关或回答质量差，如何优化？

#### 优化策略
1. **文档分块优化**
   - 调整chunk_size和chunk_overlap
   - 尝试不同分块策略
   - 在chunk前添加文件名、章节等元信息

2. **检索优化**
   - 混合检索：BM25 + 向量检索
   - 重排序（Rerank）
   - 调整match_threshold
   - 调整Top-K数量

3. **Embedding优化**
   - 选择合适的Embedding模型
   - 微调Embedding模型
   - 查询扩展（Query Expansion）

4. **Prompt优化**
   - Few-Shot示例
   - Chain of Thought
   - 清晰的指令

5. **召回质量评估**
   - 建立评估数据集
   - 持续迭代优化

### 场景5：如何实现GPU算力调度平台？

#### 问题描述
根据电价策略调度AI训练任务，合理利用GPU资源，扩大收益。

#### 系统设计
1. **核心功能**
   - 电价感知调度：实时电价，低电价时调度任务
   - 任务编排：优先级队列，依赖管理
   - 算力资源管理：收录登记自有和外部算力
   - 监控采集：Prometheus采集GPU工况
   - 外部算力对接：集成第三方算力平台

2. **调度算法**
   - 优先级队列：任务紧急程度 + 电价
   - 贪心算法：每次选最优当前任务
   - DAG拓扑排序：处理任务依赖

3. **技术实现**
   - Go后端开发
   - Prometheus监控集成
   - MySQL/Redis数据存储
   - 任务状态机管理
