# Agent 核心模块

> 一句话：一个工业级 Agent 由推理、规划、记忆、工具、上下文、工作流、错误恢复、质量判定八大模块组成。

---

## 一、模块总览

```
┌─────────────────────────────────────────┐
│            用户输入 / 多轮对话            │
└─────────────────┬───────────────────────┘
                  ▼
┌─────────────────────────────────────────┐
│  推理模块  →  规划模块  →  工作流编排      │
│  （大脑）    （蓝图）     （调度）         │
└─────────────────┬───────────────────────┘
                  ▼
┌─────────────────────────────────────────┐
│  工具调用  +  记忆查询  +  上下文管理      │
│  （手脚）    （经验）     （短期记忆）      │
└─────────────────┬───────────────────────┘
                  ▼
┌─────────────────────────────────────────┐
│  错误恢复  →  回复判定  →  最终输出        │
│  （兜底）    （质检）     （回答）         │
└─────────────────────────────────────────┘
```

---

## 二、八大核心模块

### 1. 推理模块

**作用**：理解用户意图，做逻辑推理和决策判断。

**包含能力**：
- 任务拆解
- 意图识别
- 工具选择判断
- 结果校验

**一句话**：Agent 的"大脑"，决定"想什么"。

---

### 2. 规划模块

**作用**：把复杂任务拆成可执行的子任务清单。

**输出**：
- 子任务列表
- 执行顺序
- 依赖关系
- 终止条件

**一句话**：复杂任务的"施工图纸"。

---

### 3. 工具调用模块

**作用**：根据决策调用外部工具/API/数据库。

**核心流程**：
```
选工具 → 生成参数 → 执行 → 解析结果 → 反馈给 LLM
```

**一句话**：Agent 的"手脚"，负责真实操作。

---

### 4. 记忆模块

**作用**：保存对话历史、业务知识、用户偏好。

**分层**：
| 类型 | 存储 | 作用 |
|---|---|---|
| 短期记忆 | 上下文窗口 | 当前多轮对话 |
| 长期记忆 | 向量库/数据库 | 历史对话、用户画像 |
| 业务记忆 | 知识库 | 产品规则、SOP |

**一句话**：让 Agent"记得住"，越用越聪明。

---

### 5. 上下文管理模块

**作用**：控制 Token 使用量，保证关键信息不丢失。

**常用手段**：
- 滑动窗口保留近期对话
- 摘要压缩远期历史
- 关键信息置顶
- 冗余信息过滤

**一句话**：防止上下文爆炸和遗忘。

---

### 6. 工作流编排模块

**作用**：按业务 SOP 调度多个 Skill/Plugin/子 Agent。

**支持模式**：
- 顺序执行
- 并行执行
- 条件分支
- 循环迭代

**一句话**：把原子能力组装成完整业务流程。

---

### 7. 错误恢复模块

**作用**：捕获异常、重试、兜底，保证服务不中断。

**处理场景**：
- 参数错误
- 接口超时
- 工具返回空
- 模型输出异常

**一句话**：Agent 的"保险丝"。

---

### 8. 回复判定模块

**作用**：对最终输出做质量校验。

**校验维度**：
- 是否回答用户问题
- 是否基于真实数据
- 是否合规
- 是否完整

**一句话**：最后一道质量关。

---

## 四、面试高频问题

### Q1：Agent 有哪些核心模块？

**答**：推理、规划、工具调用、记忆、上下文管理、工作流编排、错误恢复、回复判定。

### Q2：记忆模块为什么分层？

**答**：上下文窗口有限且贵，短期记忆保当前对话连贯；长期记忆用向量库存海量历史，突破窗口限制；业务记忆存规则知识，保证专业准确。

### Q3：上下文管理怎么做？

**答**：滑动窗口 + 摘要压缩 + 关键信息置顶 + 冗余过滤，必要时预留输出 Token 余量。

### Q4：错误恢复模块怎么处理工具失败？

**答**：先有限重试，再参数修正，再降级兜底，最后返回标准化异常话术，绝不编造。

### Q5：回复判定模块怎么防幻觉？

**答**：校验答案是否基于检索或工具结果，空数据时强制"不知道"，不推测不脑补。

## 五、项目口述模板

> 你项目中的 Agent 有哪些核心模块？

我项目中的 Agent 分为八大模块：推理模块负责理解意图和决策；规划模块负责复杂任务拆解；工具调用模块执行 Function Call；记忆模块保存短期和长期记忆；上下文管理模块控制 Token 和防止信息丢失；工作流编排模块按 SOP 调度多步骤执行；错误恢复模块处理超时、空结果等异常；回复判定模块做最终质量校验。这些模块形成完整的感知、思考、行动、反馈闭环。
