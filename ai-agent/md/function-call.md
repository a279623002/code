# Agent 函数调用（Function Call）

> 一句话：Function Call 让大模型从"说话"变成"办事"，能主动调用外部函数/API。

---

## 一、什么是 Function Call？

**一句话**：Function Call 是大模型输出结构化调用指令的能力，让 Agent 可以调用外部工具、查询数据库、执行代码等。

### 与普通问答的区别

| 普通问答 | Function Call |
|---|---|
| 模型直接生成答案 | 模型生成函数名和参数 |
| 依赖训练知识 | 可获取实时/私有数据 |
| 容易幻觉 | 基于真实工具结果回答 |

---

## 二、Function Call 完整链路

```
用户提问
   │
   ▼
LLM 判断是否需要工具
   │
   ├── 不需要 → 直接回答
   │
   └── 需要
        │
        ▼
   选择函数 + 生成参数
        │
        ▼
   本地参数校验
        │
        ▼
   执行函数 / API
        │
        ▼
   结果回传给 LLM
        │
        ▼
   LLM 生成最终答案
```

## 三、Function Schema 设计

### 1. Schema 是什么？

**一句话**：Schema 是工具函数的结构化说明书，告诉模型这个函数是干什么的、需要什么参数。

### 2. 标准 Schema 示例

```json
{
  "name": "get_weather",
  "description": "查询指定城市的天气情况",
  "parameters": {
    "type": "object",
    "properties": {
      "city": {
        "type": "string",
        "description": "城市名称，例如：北京"
      },
      "date": {
        "type": "string",
        "description": "日期，格式 YYYY-MM-DD，默认为今天"
      }
    },
    "required": ["city"]
  }
}
```

### 3. Schema 设计原则

| 原则 | 说明 |
|---|---|
| **描述清晰** | 说明函数能做什么、不能做什么 |
| **参数明确** | 类型、必填、取值范围写清楚 |
| **场景隔离** | 不同工具的 description 避免语义重叠 |
| **示例辅助** | 复杂参数给示例 |

## 四、多轮 Function Call

### 1. 什么是多轮调用？

**一句话**：一个任务需要连续调用多个函数，根据上一轮结果决定下一步。

### 2. 示例

```
用户：查张三的最近一笔订单金额

调用1：get_user(name="张三")
返回：{user_id: 1001}

调用2：get_orders(user_id=1001, limit=1)
返回：[{order_id: 2001, amount: 199}]

输出：张三最近一笔订单金额是 199 元
```

## 五、常见异常与处理

| 异常类型 | 原因 | 处理方案 |
|---|---|---|
| **选错工具** | 工具描述模糊 | 优化 description，加场景隔离 |
| **参数缺失** | 用户没提供 | 自动追问用户补全 |
| **参数错误** | 类型/格式不对 | 本地校验，拦截脏参数 |
| **接口超时** | 网络/服务问题 | 有限重试，失败兜底 |
| **空结果** | 查无数据 | 禁止编造，返回"未找到" |
| **死循环** | 重复调用同一工具 | 限制调用次数，拦截重复参数 |

## 六、面试高频问题

### Q1：什么是 Function Call？

**答**：Function Call 是大模型输出结构化函数调用指令的能力，让 Agent 能调用外部工具获取真实数据。

### Q2：Function Call 和 ReAct 的关系？

**答**：ReAct 是思考框架，Function Call 是执行手段。ReAct 的 Action 通常就是 Function Call。

### Q3：Schema 为什么重要？

**答**：Schema 是模型理解工具、生成正确参数的唯一依据。描述不清会导致选错工具、参数错误。

### Q4：怎么防止模型死循环调用？

**答**：限制最大调用次数、拦截相同函数+相同参数的重复调用、明确终止条件。

### Q5：工具返回空数据怎么办？

**答**：强制模型不推测、不编造，返回标准化"未查询到数据"的兜底话术。

## 七、项目口述模板

> 你项目中 Function Call 是怎么落地的？

我在项目中用 Function Call 实现 Agent 的工具调用能力。首先为每个工具设计标准 Schema，包含名称、功能描述、参数类型、必填字段和示例；Agent 接收到用户请求后，LLM 判断是否需要调用工具，并生成结构化调用参数；本地做参数校验后执行函数，将结果回传给 LLM，由 LLM 生成最终答案。针对多步骤任务，支持多轮 Function Call 迭代。同时做了异常兜底：参数错误自动追问、接口超时有限重试、空结果禁止编造、死循环通过次数限制和重复调用拦截来避免。

---

## 八、一句话总结

- **Function Call = 模型生成函数调用指令**
- **Schema 是工具说明书，决定调用准确率**
- **多轮调用 = 复杂任务的拆解执行**
- **异常兜底：参数校验、重试、防幻觉、防死循环**
