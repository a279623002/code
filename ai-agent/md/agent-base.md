# Agent 基础原理

> 一句话：Agent = 大模型（大脑）+ 工具（手脚）+ 记忆（经验），能自主思考、调用工具、完成任务。

---

## 一、Agent 是什么？

**一句话**：Agent 是基于大模型的自主任务执行体，不只是聊天，还能理解需求、拆解任务、调用工具、迭代纠错，最终把事做完。

### Agent 与普通大模型的区别

| 对比 | 普通大模型 | Agent |
|---|---|---|
| 交互方式 | 问一句答一句 | 主动规划、多轮执行 |
| 知识来源 | 训练数据 | 训练数据 + 实时工具/数据库 |
| 能否行动 | 不能 | 能调用 API、查库、执行代码 |
| 纠错能力 | 弱 | 能根据结果调整策略 |
| 适用任务 | 简单问答 | 复杂业务流程 |

## 二、Agent 核心三要素

```
        用户输入
           │
           ▼
    ┌─────────────┐
    │   LLM 大脑   │  ← 理解、推理、决策
    │  思考/规划   │
    └──────┬──────┘
           │
     ┌─────┴─────┐
     ▼           ▼
  工具调用     记忆查询
  （手脚）     （经验）
     │           │
     └─────┬─────┘
           ▼
      外部结果反馈
           │
           ▼
    ┌─────────────┐
    │  迭代/输出   │
    └─────────────┘
```

| 要素 | 作用 | 例子 |
|---|---|---|
| **大脑（LLM）** | 理解意图、拆解任务、推理决策 | GPT、Claude、DeepSeek |
| **手脚（工具）** | 执行真实操作，获取实时数据 | 查天气、查订单、执行 SQL |
| **记忆（Memory）** | 保存上下文和历史，保证多轮连贯 | 向量库、数据库、缓存 |

## 三、Agent 工作流

```
用户提问
   │
   ▼
理解意图 ──► 判断是否需要工具
   │
   ├── 不需要 ──► 直接回答
   │
   └── 需要 ──► 选择工具 ──► 生成参数 ──► 执行工具
                           │
                           ▼
                    观察返回结果
                           │
              ┌────────────┴────────────┐
              ▼                         ▼
        任务已完成                   任务未完成
              │                         │
              ▼                         ▼
        生成最终答案              继续调用工具
```

## 四、核心推理策略

| 策略 | 核心思想 | 适用场景 |
|---|---|---|
| **CoT** | 让模型分步思考 | 数学、逻辑推理题 |
| **ReAct** | 思考 + 行动交替 | 多轮工具调用 |
| **Plan-Execute** | 先规划，再执行 | 复杂长流程任务 |

### 简单对比

```
CoT：          问题 → 思考 → 思考 → 答案（只做脑内推理）
ReAct：        问题 → 思考 → 行动 → 观察 → 思考 → 行动 → 答案
Plan-Execute： 问题 → 全局规划 → 分步执行 → 汇总答案
```

## 五、面试高频问题

### Q1：什么是 Agent？和 Chatbot 有什么区别？

**答**：Agent 是具备自主规划、工具调用、记忆能力的智能体，能主动完成复杂任务；Chatbot 主要是被动对话，依赖模型已有知识，不能调用外部工具。

### Q2：Agent 为什么需要工具？

**答**：大模型有三大局限：知识有时效性、无法访问私有数据、容易幻觉。工具能弥补这些短板，让 Agent 获取实时、真实、私有数据。

### Q3：Agent 三要素是什么？

**答**：大脑（LLM 推理）、手脚（工具执行）、记忆（上下文/长期记忆）。三者协作完成感知、思考、行动、反馈的闭环。

### Q4：ReAct 和 CoT 的区别？

**答**：CoT 只做思维推理，不调用工具；ReAct 是思考 + 行动 + 观察的循环，能调用外部工具并迭代。

### Q5：Agent 怎么保证多轮不跑偏？

**答**：通过上下文管理、状态机、记忆机制、话题锁定。关键信息置顶，冗余信息裁剪，需求变更时覆盖旧参数。

## 六、项目口述模板

> 请介绍你做的 Agent 项目。

我在项目中落地的 Agent 体系由大模型作为推理大脑，Function Call 作为工具执行手段，向量库和数据库作为长期记忆。整体工作流是：用户输入先进入意图理解模块，判断是否需要调用工具；如果需要，Agent 会选择合适工具、生成参数、执行并观察结果；如果任务没完成，会继续迭代；最后基于真实结果生成答案。针对简单任务用 ReAct 快速迭代，针对复杂长流程任务用 Plan-Execute 先规划再执行，保证任务可控、可追溯。

---

## 七、一句话总结

- **Agent 不是模型，是用模型驱动的任务执行体**
- **三要素：大脑、手脚、记忆**
- **核心流程：理解 → 决策 → 行动 → 观察 → 迭代**
- **简单任务 ReAct，复杂任务 Plan-Execute**
