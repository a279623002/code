# Agent 四大工程难题

> 一句话：Agent 线上最常见的四大问题是幻觉、上下文干扰、长文本退化、工具失效，对应四种工程解法。

---

## 一、模型幻觉

### 1. 现象

模型脱离真实数据，编造事实、数据、流程、政策。

### 2. 原因

- 模型训练目标偏向"说得通"而非"绝对真实"
- 检索召回不全或工具无返回
- Prompt 缺少忠实度约束
- 上下文混入噪声

### 3. 解决方案

| 层级 | 方案 |
|---|---|
| 源头 | 优化 RAG 召回，补齐真实资料 |
| 过程 | Prompt 强制约束：无依据不编造 |
| 后置 | 答案与检索/工具结果校验 |
| 模型 | 幻觉 BadCase 微调 |

---

## 二、上下文干扰

### 1. 现象

多轮对话中话题跑偏、新旧需求冲突、答非所问。

### 2. 原因

- 历史上下文冗余噪声多
- 多话题交叉干扰
- 用户修改需求后旧参数未覆盖

### 3. 解决方案

| 方案 | 说明 |
|---|---|
| 智能降噪 | 过滤过期、重复、无关信息 |
| 话题锁定 | 当前主话题屏蔽无关历史 |
| 结构化组装 | 区分当前需求、历史、检索结果 |
| 需求覆盖 | 用户更新需求时标记旧参数失效 |

---

## 三、长文本退化

### 1. 现象

上下文过长后，模型遗忘关键信息、细节遗漏、推理失效。

### 2. 原因

- 注意力机制对远距离信息关联弱
- 信息密度低，关键内容被淹没
- 全量历史直接灌入

### 3. 解决方案

| 方案 | 说明 |
|---|---|
| 上下文提纯 | 摘要压缩，保留核心 |
| 按需加载 | 向量记忆召回相关内容 |
| 滑动窗口 | 保留近期对话，淘汰久远内容 |
| 关键信息置顶 | 核心规则固定放 Prompt 头部 |

---

## 四、工具调用失效

### 1. 现象

该调不调、调错工具、参数错误、超时失败、死循环。

### 2. 原因

- 工具描述不清晰
- 用户输入模糊缺参
- 模型决策偏差
- 缺少校验纠错机制

### 3. 解决方案

| 问题 | 方案 |
|---|---|
| 选错工具 | 优化 Schema、场景隔离 |
| 参数错误 | 本地校验、缺参追问 |
| 接口超时 | 有限重试、降级兜底 |
| 死循环 | 调用次数上限、重复调用拦截 |
| 空结果 | 禁止编造，标准化兜底 |

---

## 五、面试高频问题

### Q1：Agent 最大的工程问题是什么？

**答**：幻觉、上下文干扰、长文本退化、工具调用失效。

### Q2：怎么解决幻觉？

**答**：RAG 补齐真实资料 + Prompt 强制约束 + 后置真实性校验 + 模型微调。

### Q3：多轮对话跑偏怎么办？

**答**：上下文降噪、话题锁定、需求增量覆盖、结构化组装。

### Q4：长文本怎么处理？

**答**：分片、摘要、按需向量召回、滑动窗口、关键信息置顶。

### Q5：工具调用失败怎么兜底？

**答**：参数校验、有限重试、降级提示、禁止编造、死循环拦截。

## 六、项目口述模板

> 你在 Agent 项目中遇到过哪些问题？怎么解决的？

我在 Agent 落地中重点解决了四大工程问题。针对幻觉，通过 RAG 检索增强 + 忠实度 Prompt 约束 + 后置校验，让模型只基于真实数据回答；针对上下文干扰，通过话题锁定、上下文降噪和需求增量覆盖，保证多轮对话紧跟用户最新需求；针对长文本退化，通过分片、摘要、向量按需召回和滑动窗口管理上下文；针对工具调用失效，通过 Schema 优化、参数校验、有限重试、死循环拦截和空结果兜底，保证工具链路稳定。

---

## 七、一句话总结

- **幻觉：检索 + 约束 + 校验**
- **上下文干扰：降噪 + 锁定 + 覆盖**
- **长文本退化：摘要 + 按需 + 置顶**
- **工具失效：校验 + 重试 + 拦截 + 兜底**
